# AntesCargo

Статическое веб-приложение для логистики, рейсов и финансовых сводок на SQLite в браузере.

## Быстрый старт

Откройте `index.html` через локальный сервер:

```powershell
node scripts/dev-server.mjs
```

Адрес:

```text
http://127.0.0.1:4173/
```

При первом открытии задайте пароль администратора в форме входа. Пароль сохраняется как PBKDF2-SHA256-хэш в локальном хранилище браузера и не появляется в файлах проекта.

Для GitHub Pages достаточно загрузить в репозиторий:

```text
index.html
src/
scripts/
README.md
```

В настройках GitHub Pages выберите публикацию из корня репозитория.

## Что реализовано

- SQLite через `sql.js` в браузере.
- Сохранение базы в IndexedDB.
- Экран входа: администратор и водитель.
- Пароль администратора создается локально при первом запуске и не хранится в репозитории.
- Вход водителя по телефону из карточки водителя.
- Скачивание базы в `.sqlite`.
- Экспорт общего отчета в Excel `.xlsx`.
- CSV с UTF-8 BOM и `;` для Excel.
- Финансы: общий, топливо, Платон, транспондер.
- Общая сводка по водителям, машинам, заявкам и рейсам.
- Нормализация телефона, водителя, госномера и транспондера.
- История закреплений машины за водителем.
- Загрузка CSV/XLSX отчетов топлива, Платона и транспондера.
- Дедупликация операций по UID/SHA-256.
- Кабинет водителя: вход по телефону, загрузка, выгрузка, забытый рейс, заявки.
- Non-blocking loader снизу справа для долгих операций.

## Важное ограничение GitHub Pages

GitHub Pages не запускает FastAPI и серверный SQLite. Поэтому текущая версия хранит SQLite локально в браузере пользователя. Загруженные отчеты, рейсы, заявки и финансы не попадают в GitHub и не видны другим посетителям сайта, пока вы сами не загрузите `.sqlite`/Excel-файл в репозиторий или не передадите его.

Публичный репозиторий содержит только код. Реальные стартовые машины, водители, госномера, телефоны и транспондеры в код не зашиты.

Важно: локальный пароль администратора защищает интерфейс на устройстве, но не превращает GitHub Pages в защищенный многопользовательский сервер. Любой посетитель публичной страницы может открыть свою пустую локальную копию приложения в своем браузере. Ваши данные он не увидит, потому что они лежат в IndexedDB вашего браузера.

Для полноценной многопользовательской работы нужен backend:

```text
FastAPI + SQLModel/SQLAlchemy + SQLite/Postgres + Docker
```

Frontend из этого проекта можно оставить как основу интерфейса.
